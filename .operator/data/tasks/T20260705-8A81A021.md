# Split weekly-metrics-stage.ts under the pipeline line cap

## Problem
`engine/pipeline/composers/weekly-metrics-stage.ts` is ~215 code lines
(excluding logging/comments/JSDoc) — marginally over the `engine/pipeline/**`
≤200 cap. The densest section is `buildWeeklyMetricsAfterAgent` (`235-335`):
report-file cleaning/writing (`283-296`) plus supplementary AOP application
(`299-325`).

## Solution
Extract the report-output path (output cleaning via `stripPreamble` /
`stripCodeFences` / `parseAgentOutput`, report-file write, and the
supplementary `applyAgentEvents` call) into a small helper — either
`engine/pipeline/composers/_shared/metrics-report.ts` (with a colocated test)
or fold the AOP application into an existing primitive if one fits.
`buildWeeklyMetricsAfterAgent` then orchestrates: verdict guard → extracted
writer → summary.

Behavior-preserving: failure-marker handling and PR-body selection stay put.
Keep all logging lines intact.
