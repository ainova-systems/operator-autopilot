# Redact structured log `data` and `child` bindings in `wrapPino`

## Problem
`wrapPino` (`engine/logging/logger.ts:33-67`) redacts only the message
string (`redactString(msg)`) and passes the structured `data` object to pino
unmodified on all four levels (`:37` debug, `:44` info, `:51` warn, `:58`
error). The pino instances carry no `redact` option (`:79-80`, `:139`, `:145`),
so there is no serializer-level fallback. The purpose-built recursive scrubber
`redactValue` (`engine/logging/redact.ts:51-66`) is imported by no runtime
module — only by its own test — so structured-field redaction never runs.

Concrete leak: `engine/agents/providers/cli.ts:208-214` logs a WARN whose
`data` carries `stderr`/`stdout` slices (1000 chars each) from the agent
subprocess on a non-zero exit. That output routinely echoes `Authorization:
Bearer …` headers and auth-error dumps, all persisted verbatim. Any token in a
`data` field (error `.cause` strings, captured HTTP bodies, git remote URLs)
reaches the sink in cleartext.

## Solution
In `engine/logging/logger.ts`:
- In `wrapPino`, pass every `data` object through `redactValue` before handing
  it to pino, on debug/info/warn/error. `redactValue` on an object returns a
  fresh scrubbed object — cast its result to `Record<string, unknown>` at the
  pino boundary (do NOT use `as any`; that is forbidden by `typescript.md`).
- In `child(bindings)`, pass `bindings` through `redactValue` before calling
  `pinoLogger.child(...)` so bound context fields are scrubbed too.
- Keep the existing `redactString(msg)` for the message.
- Add the `redactValue` import from `./redact.js` (alongside `redactString`).
- Update the doc comments at `:19` and `:31` to state that structured fields
  (and child bindings) are redacted, not only the message.

Keep the change minimal and within the 300-line cap for `engine/logging/`.

## Regression test (failing-first, required)
Add to `engine/logging/logger.test.ts` a test whose name describes the bug
scenario (e.g. "redacts a ghp_ token in structured data fields (CLI-exit
stderr/stdout leak)"). Capture pino output via an injected stream (the file
already has a `fakeOut` helper and `createConsole` accepts an `out` stream),
log a WARN with `data` modelling the `cli.ts` case — `{ stderr:
"Authorization: Bearer <token>", stdout: "ghp_<30+ chars>" }` — and assert the
serialized output contains `[REDACTED]` and NOT the raw token. Add a second
assertion that a `child({ ... })` binding containing an `sk-ant-…` token is
also redacted. The test MUST fail on the pre-fix logger and pass after.

## Affected Files
