# Split discovery-iteration-stage.ts under the pipeline line cap

## Problem
`engine/pipeline/composers/discovery-iteration-stage.ts` is ~250 code lines
(excluding logging/comments/JSDoc) — over the `engine/pipeline/**` ≤200 cap.
`runSingleAnalyzer` (`136-238`, ~100 lines) is a self-contained per-analyzer
runner: it builds the `AgentRunInput`, invokes `agentRuntime.run`, routes
output through `applyAgentEvents`, and syncs the resulting child item.

## Solution
Extract `runSingleAnalyzer` (and its `outputExcerpt` helper, `128-134`) into
a dedicated module. Because it calls `AgentRuntime.run` and `applyAgentEvents`
from a stage path, the extracted file must remain inside the primitives/stage
boundary — place it at `engine/pipeline/primitives/analyzer-run.ts` (colocated
`analyzer-run.test.ts`, ≥95% coverage as a primitive) or, if kept as composer
glue, `engine/pipeline/composers/_shared/analyzer-run.ts`. Prefer the
primitives location since it wraps `AgentRuntime.run`. `buildDiscoveryIterationBeforeAgent`
then calls the extracted function through `iterateBestEffort` as today.

Behavior-preserving: analyzer ordering, known-item skipping, and rejection
sub-flow wiring are unchanged. Keep every log line intact.

## Affected Files
