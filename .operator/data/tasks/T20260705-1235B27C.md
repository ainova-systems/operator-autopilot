# Replace the `node:fs/promises` mock in `workspace.test.ts` with real temp directories

## Problem
`engine/infra/workspace.test.ts:12-15` mocks `node:fs/promises`
(`access`, `mkdir`), violating `intelligence/rules/typescript.md` §REQUIRED
("Real temp directories for filesystem tests via `fs.mkdtemp`. Never mock
`fs`"). The clone-vs-existing branch is driven by `mockAccess.mockRejectedValue`
/ `mockAccess.mockResolvedValue` (`:81`, `:103`) instead of a real directory,
and every assertion is an `execFile` call-shape check
(`mockExecFile.mock.calls.filter(...)`, e.g. `:92-99`, `:110-122`). A break in
`isGitRepo`'s `.git` probe (`workspace.ts:123-130`, `:148`), in the path join,
or in `mkdir(parentDir, { recursive: true })` (`workspace.ts:180`) would leave
every test green. This is the safety-relevant git-workspace module
(2026-04-13 non-fast-forward lineage).

## Solution
Remove the `vi.mock("node:fs/promises", ...)` block entirely. Let `access` and
`mkdir` run against a real base directory created per-test with
`fs.mkdtemp(join(tmpdir(), "workspace-test-"))` and removed in `afterEach` with
`rm(dir, { recursive: true, force: true })`, mirroring
`engine/infra/git.test.ts:15-29` and `engine/infra/workspace-init.test.ts:38-40`.
Pass the real `mkdtemp` dir as `workspaceBaseDir` in place of `"/base"`.
Keep the `node:child_process` (`execFile`) fake — spawning real `gh`/`git` is
out of scope; only the fs boundary must be real.

- Drive the "clones when workspace does not exist" case off an **empty** real
  base dir (no `<id>/.git`) so `isGitRepo` returns false, and assert on disk
  that the parent directory was created by `mkdir` (in addition to, or instead
  of, the `gh clone` call-shape check).
- Drive the "fetches and resets when workspace already exists" case by
  pre-creating `<base>/<id>/.git` (compute the workspace path via the same
  `workspacePath` helper the source uses, or by mirroring it) so `isGitRepo`
  returns true.
- Preserve the existing error-path and checkout-fallback coverage
  (`ConfigError` on missing token, `WorkspaceError` on git failure, `-B`
  fallback, aborted signal), rewired onto the real-dir setup.

## Affected Files
- `engine/infra/workspace.test.ts:12-15` — remove the `node:fs/promises` mock
- `engine/infra/workspace.test.ts:80-247` — rewire `workspaceEnsure` cases onto
  a real `mkdtemp` base dir; branch clone-vs-existing on a real `.git` marker;
  assert on-disk `mkdir` effect for the clone case
- `engine/infra/workspace.test.ts` (top) — add `mkdtemp`/`rm`/`tmpdir` imports
  and `beforeEach`/`afterEach` temp-dir lifecycle, following `git.test.ts`

## Acceptance Criteria
- No `vi.mock("node:fs/promises")` (or any `fs` mock) remains in
  `engine/infra/workspace.test.ts`
- The clone-vs-existing branch is selected by the real on-disk presence/absence
  of `<workspace>/.git`, created under an `fs.mkdtemp` base dir
- At least one assertion checks a real on-disk effect (parent/workspace
  directory created), not only `execFile` call shapes
- Temp dirs are removed in `afterEach` (no leaked directories)
- `npm run typecheck && npm run lint && npm test` pass; coverage on
  `engine/infra/workspace.ts` stays >=90%
