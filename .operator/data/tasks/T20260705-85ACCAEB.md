# Fix improver.md AOP EMIT syntax to the fenced form the parser accepts

## Problem
The bundled improver prompt documents AOP emission using colon-key
pseudo-syntax (`EMIT: status-update`) that the AOP text-block parser
never recognizes. The parser accepts ONLY fenced markers:
- `engine/pipeline/primitives/agent-output-protocol.ts:41` —
  `START_MARKER = /^===\s*EMIT\s+([a-zA-Z][a-zA-Z0-9-]*)\s*===\s*$/`
- `:42` — `END_MARKER = /^===\s*END EMIT\s*===\s*$/`

The improver IS wired through this parser: `stages.yaml:226-258` runs the
`retrospective` stage as `agentRole: improver` via composer `weekly-metrics`;
`engine/engine/stage-handlers.ts:503` builds `new TextBlockEventStream(parseAgentOutput)`
and `engine/pipeline/composers/weekly-metrics-stage.ts:300` feeds the improver's
stdout to `applyAgentEvents` through that stream. A line like `EMIT: status-update`
fails `START_MARKER`, is treated as free text, and — lacking a `---` fence — does
not even trip the `raw-frontmatter-leak` guard. Result: `parseResult.events` is
empty, `applyAgentEvents` returns `statusUpdates: [], childItems: []`, and
`weekly-metrics-stage.ts:311` logs nothing. Goal 2 task cancellation, Goal 3
duplicate detection, and every "Status Reconciliation" the improver reports are
emitted-but-dropped on a successful run — while the prompt forbids the only
working fallback (`improver.md:109-110`, "never bypass it by editing frontmatter
directly"). Every other prompt (`supervisor.md`, `planner.md:86`, `analyst.md:92`)
already uses the fenced form.

## Solution
Rewrite the three `EMIT:` occurrences in `engine/content/prompts/agents/improver.md`
to fenced blocks, matching `supervisor.md:70-75` / `planner.md:86-107`. The
status-update example must become:
```
=== EMIT status-update ===
target: F20260416-0002
status: in-progress
reason: "children ... already merged on develop; parent never advanced"
=== END EMIT ===
```
and the prose references at `:111-112` must name `EMIT child-item` / `EMIT body-update`
as fenced blocks (drop the `EMIT:` colon form entirely). Runtime code is correct —
do NOT change the parser, schema, or wiring.

Add a failing-first regression contract test (colocate with the parser test at
`engine/pipeline/primitives/agent-output-protocol.test.ts`, or a new colocated
`*.test.ts`) that:
1. Reads the actual `engine/content/prompts/agents/improver.md` via `resolveContentPath`.
2. Extracts the status-update example block and feeds it through `parseAgentOutput`.
3. Asserts exactly one `status-update` `AgentEvent` with zero `severity: "error"` diagnostics.
The test name must describe the bug scenario (e.g.
"improver.md status-update example parses as a fenced AOP status-update event").
It must fail on the pre-fix `EMIT:` colon text and pass after the rewrite.

## Affected Files
- `engine/content/prompts/agents/improver.md:100-105` — replace the `EMIT: status-update`
  YAML block with a `=== EMIT status-update === ... === END EMIT ===` fenced block.
- `engine/content/prompts/agents/improver.md:111-112` — rewrite the prose to reference
  `EMIT child-item` / `EMIT body-update` as fenced blocks, not `EMIT: ...`.
- `engine/pipeline/primitives/agent-output-protocol.test.ts` — add the contract
  regression test reading improver.md (or a new colocated test file).
