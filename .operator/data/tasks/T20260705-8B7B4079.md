# Split aop-planner-stage.ts under the pipeline line cap

## Problem
`engine/pipeline/composers/aop-planner-stage.ts` is 561 total / ~425 code
lines (excluding logging/comments/JSDoc) — well over the `engine/pipeline/**`
≤200 cap.
`buildAopPlannerBuildPR` (`270-354`) carries three PR-body renderings
(rejection body `288-302`, catch-up body `315-325`, plan-path template call
`332-342`), and `buildTerminalComment` (`552-561`) is a standalone string
helper.

## Solution
Extract the PR-title/body/commit-message construction for the rejection,
catch-up, and plan paths into
`engine/pipeline/composers/_shared/planner-pr-body.ts` (pure functions taking
the item + rejection/catch-up context, returning `{title, body,
commitMessage}`), with a colocated `planner-pr-body.test.ts`. Optionally move
`buildTerminalComment` there too. `buildAopPlannerBuildPR` becomes a thin
dispatcher that reads scratch and calls the extracted builder.

Behavior-preserving: scratch handling, verdict routing, and label transitions
are unchanged. Keep all logging lines in place.

## Affected Files
- `engine/pipeline/composers/aop-planner-stage.ts:270-354` — reduce `buildAopPlannerBuildPR` to a dispatcher; import from `_shared`
- `engine/pipeline/composers/aop-planner-stage.ts:552-561` — optionally move `buildTerminalComment`
- `engine/pipeline/composers/_shared/planner-pr-body.ts` — new: extracted PR-body builders
- `engine/pipeline/composers/_shared/planner-pr-body.test.ts` — new: colocated test
- `engine/pipeline/composers/aop-planner-stage.test.ts` — update imports if needed

## Acceptance Criteria
- `aop-planner-stage.ts` is ≤200 code lines (logging/comments/JSDoc excluded)
- New `_shared` module has a colocated test at ≥90% coverage; existing composer behavior tests pass unchanged
- Rejection/catch-up/plan PR bodies are byte-identical to pre-refactor output (assert in the new test)
- Every pre-existing INFO/WARN/ERROR line is preserved
- `ts-prune`/`knip` clean; named exports only
- `npm run typecheck && npm run lint && npm test` pass
