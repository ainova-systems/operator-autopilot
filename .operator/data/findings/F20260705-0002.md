**Pattern**: Retired-version branding surviving in live user-facing output
**Domain**: engine/cli.ts

**Impact**: `engine/cli.ts` line 2 documents the module as "CLI argument parser for Operator V3" and `printHelp` (line 96) prints "Operator V3 — AI-powered SDLC engine" as the first line of `--help` output. The engine is the v5 rebuild (package version 0.5.x); every operator running `--help` is told they are using a two-generations-old product. Stale version literals in user-facing output erode trust in the rest of the documentation and will silently survive future rebuilds exactly as this one survived v4 → v5.

**Fix**: Replace the hardcoded generation label: read the product name and version from `package.json` (single source of truth) and render the banner from them, or drop the generation entirely ("Operator — AI-powered SDLC engine"). Update the module JSDoc header the same way. Sweep live code (`engine/`, `packages/`, `app/`, excluding historical docs) for remaining "V3"/"V4" references in the same change.

**Acceptance Criteria**:
- [ ] `--help` output contains no retired generation label; the version shown matches `package.json`
- [ ] No "Operator V3"/"Operator V4" strings remain in live source (historical docs excluded)
