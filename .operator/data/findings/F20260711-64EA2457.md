**Severity**: medium
**Priority**: 4
**Files Affected**: 2

**Pattern**: v5 observability mandate violation (`intelligence/rules/typescript.md` §REQUIRED "every catch-and-continue branch MUST warn with the reason"; `migration.md` observability BLOCKER gate). Two primitive-layer catch-and-continue branches around KV writes swallow the error silently.
**Domain**: engine/pipeline/primitives (route-verdict, observe-status)

**Impact**: These are not in `GitHubVCS` (already tracked by the logger task) — they are in the orchestrator primitives that own KV state:
- `engine/pipeline/primitives/route-verdict.ts:160-162` — the `catch {}` around the `kv.put("work-items", …)` reconciliation write (status, `statusReason`, `attemptCount`, drift) at line 143 is empty. `RouteVerdictDeps` (line 43) carries no `Logger`, so a failed reconcile is invisible: a lost `attemptCount` increment silently defeats the bounded-iteration guard, and status/drift updates vanish with zero log evidence — exactly the "I can't see what the engine did" failure v5 exists to prevent.
- `engine/pipeline/primitives/observe-status.ts:239-241` — the `catch {}` around `recordTerminalPRStates(...)` (pr-state cache write) is empty; `ObservePRStateDeps` (line 186) carries no `Logger` (the `log?` at line 277 belongs to the separate `ObserveChecksDeps`).

Both paths complete "successfully" while a write failed, with no audit trail — a silent-success bug per the mandate.
**Fix**: Add `readonly log?: Logger` to `RouteVerdictDeps` and `ObservePRStateDeps`, thread it from the composition root, and replace each empty `catch` with `catch (err) { deps.log?.warn("<what failed> — continuing best-effort", { …ids, error: errorMessage(err) }) }`, preserving the best-effort (non-throwing) behavior. Keep `.cause` via `errorMessage`/error metadata.

**Acceptance Criteria**:
- [ ] Both `catch` branches emit a WARN with the failing operation + identifiers (workItemId / prNumber) and the error reason
- [ ] `RouteVerdictDeps` and `ObservePRStateDeps` expose an optional `Logger`, wired from `entry.ts`
- [ ] Regression tests assert a WARN is logged when the KV write rejects (fails on pre-fix code), keeping primitive coverage ≥95%
- [ ] Build passes after fix
