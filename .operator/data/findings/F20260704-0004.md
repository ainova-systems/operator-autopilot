**Severity**: medium
**Priority**: 4
**Files Affected**: 2

**Pattern**: Agent output contract drift — the parser expects a structural block the role prompt does not require
**Domain**: engine/content/prompts/agents, engine/agents

**Impact**: A successful supervisor run on PR #1 (provider: cursor) produced `WARN agent output missing ## Execution Summary block — using synthesized fallback`. The run succeeded, so the contract gap is invisible in outcomes — but the synthesized fallback replaces the agent's own summary with a generated one, degrading execution-history quality (reviewer summaries, retrospective inputs), and the recurring WARN is unowned noise. Either the supervisor prompt fails to require the block that the output parser expects, or the block is genuinely optional and the WARN overstates the condition.

**Fix**: Decide the contract explicitly. If the block is required: state it in the supervisor role prompt (and any other role whose output feeds the same parser) with the exact heading, and keep the WARN for genuine violations. If it is optional: downgrade the log to DEBUG and document the fallback as a designed path. Align every role prompt whose output is parsed for this block.

**Acceptance Criteria**:
- [ ] The supervisor prompt and the output parser agree on whether `## Execution Summary` is required
- [ ] A successful supervisor run emits either the block or no WARN (per the decided contract)
- [ ] A test covers both the block-present and block-absent parsing paths
- [ ] Build passes after fix
