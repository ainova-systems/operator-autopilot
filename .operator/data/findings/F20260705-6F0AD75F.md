**Severity**: medium
**Priority**: 4
**Files Affected**: 2

**Pattern**: Forbidden `fs` mocking in filesystem tests — `intelligence/rules/typescript.md` §REQUIRED ("Real temp directories for filesystem tests via `fs.mkdtemp`. Never mock `fs`") and the test-strategy Test-hygiene rule ("Mocked `fs`. Filesystem tests use real temp dirs via `fs.mkdtemp` — mocking `fs` is forbidden").
**Domain**: engine/infra, engine/agents

**Impact**: Two test files mock `node:fs/promises` instead of exercising the real filesystem boundary:
- `engine/infra/workspace.test.ts:12` — `vi.mock("node:fs/promises", () => ({ access: vi.fn(), mkdir: vi.fn() }))`, combined with a mocked `node:child_process` (`:8`). Every assertion is a call-shape check (`mockExecFile.mock.calls.filter(...)` matching argument substrings like `"fetch"`, `"reset"`, `"-B"`). No test verifies real directory creation, real branch/checkout state, or that the workspace layer actually reaches the intended on-disk result — so a break in path joins, mkdir recursion, or the clone-vs-existing branch decision would leave every test green. This is the safety-relevant git-workspace module (the 2026-04-13 non-fast-forward lineage), and its own sibling `engine/infra/git.test.ts` already drives **real** git against an `fs.mkdtemp` repo, proving the correct pattern is available here.
- `engine/agents/runtime.test.ts:17` — mocks `writeFile`/`unlink` for the agent system-prompt temp file, so temp-file lifecycle (path correctness, cleanup/leak on the error path) is never actually exercised.

Mocking `fs` makes these tests assert call shapes rather than behavior; they would survive a real behavioral break at the filesystem boundary, which is precisely the failure class the rule bans. The convention is well established — 42 test files (e.g. `engine/infra/workspace-init.test.ts`, `engine/infra/git.test.ts`, every touched primitive) already use `fs.mkdtemp`.

**Fix**: Replace the `node:fs/promises` mocks with real `fs.mkdtemp` temp directories, mirroring `engine/infra/git.test.ts` and `workspace-init.test.ts`. In `workspace.test.ts`, let `access`/`mkdir` run against a real temp base dir (git subprocess may still be faked, but the clone-vs-existing decision should branch on a real directory that the test creates or omits). In `runtime.test.ts`, point the system-prompt temp-file write/unlink at a real temp dir and assert the file is created and then removed (including on the failure path). Remove the `node:fs/promises` `vi.mock` calls entirely.

**Acceptance Criteria**:
- [ ] No `vi.mock("node:fs/promises")` (or any `fs` mock) remains in `engine/infra/workspace.test.ts` or `engine/agents/runtime.test.ts`
- [ ] `workspace.test.ts` drives the clone-vs-existing branch off a real `fs.mkdtemp` directory and asserts on-disk effects, not only `execFile` call shapes
- [ ] `runtime.test.ts` asserts the system-prompt temp file is created and cleaned up (including on the error path) against a real temp dir
- [ ] Build passes after fix
