**Severity**: medium
**Priority**: 4
**Files Affected**: 1 (`README.md`)

**Pattern**: Docs describing retired behavior / referencing a file that no longer exists — a README section invoking a workflow file and command that are gone, and describing a superseded automation model (staleness.md §"Docs describing retired behavior" + §"Broken internal links or references to files that were moved, deleted, or extracted").
**Domain**: README.md (front-door documentation) ↔ `.github/workflows/`, `deployment/`

**Impact**: `README.md:74` ("## Automation status") tells contributors and operators: *"The `orchestrator` GitHub Actions workflow runs on its 5-minute cron and on manual dispatch (`gh workflow run orchestrator.yml`). The workflow builds the engine, runs one `--once` cycle against every repo in `config/repos.yaml`, and surfaces the summary in the Actions UI."* Every concrete claim there is false against the current tree:
- `.github/workflows/` holds only `tests.yml` (typecheck/lint/test — `tests.yml:40-49`) and `build-image.yml` (build + `deployment/deploy.sh` — `build-image.yml:107`). There is no `orchestrator.yml`; `git log --all -- .github/workflows/orchestrator.yml` returns nothing, so it never existed. Running the documented `gh workflow run orchestrator.yml` errors out.
- No workflow runs "one `--once` cycle against every repo in `config/repos.yaml`". The only workflow `cron` is the nightly image rebuild `17 4 * * *` (`build-image.yml:31`). The "5-minute cron" is the engine's internal per-stage KV `schedule: "*/5 * * * *"` (`docs/architecture-v5.md:470`, `docs/workflow.md:338`), unrelated to GitHub Actions.
- The real deployment model is an always-on daemon container (`deployment/README.md:3` "Run the Operator engine as a single, always-on daemon container"; `deployment/docker-compose.yml:1`), built and redeployed by `build-image.yml` via a self-hosted runner — not a CI cron that surfaces summaries in the Actions UI.

A reader following the README front door is pointed at a command that fails and an automation architecture that was replaced, which is exactly the doc-drift-as-defect class `intelligence/rules/dev-context-engineering.md` forbids.

**Fix**: Rewrite the `## Automation status` section to match the single source of truth — the actual workflow files plus `deployment/README.md` / `docs/deployment.md`. Describe the two real workflows by name (`tests.yml` as the CI gate; `build-image.yml` as the nightly `17 4 * * *` image rebuild that redeploys the always-on daemon through the self-hosted runner), drop the `orchestrator.yml` / `gh workflow run orchestrator.yml` / "5-minute cron over every repo in the Actions UI" claims, and link to `docs/deployment.md` for the deployment reference. Keep the observability sentence (the `@operator/app` UI pointer is still accurate).
